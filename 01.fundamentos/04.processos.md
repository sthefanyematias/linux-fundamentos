# Gerenciamento de Processos

### Listar processos

1. `ps aux`
   - apresenta uma **relação completa dos processos em execução no sistema** para todos os usuários, exibindo o estado atual, com informações como o usuário responsável, consumo de CPU/ memória, PID (identificador do processo) e o comando completo, sendo essencial para monitoramento e depuração, permitindo identificar processos que consomem muitos recursos ou que estão sendo executados sem um terminal associado.
   - **ps**: *process status* — mostra o estado dos processos
   - **a** (*all*): exibe processos de todos os usuários 
   - **u**: mostra a saída em formato detalhado, incluindo usuário, cpu, memória, etc.
   - **x**: inclui processos que não estão associados a um terminal
  
```bash

ps aux

```

2. `top` (Top Processes)
   - comando que fornece uma **visão dinâmica e atualizada do sistema**, exibindo o consumo de cpu e memória, além da lista de processos em execução, seus recursos utilizados e níveis de prioridade, sendo essencial para acompanhar o desempenho e identificar gargalos no sistema operacional.

```bash

top

```

> **Nota:** o `htop` é a versão colorida e interativa do `top`.


### Localizar processos

1. `pgrep` (Process Grep)
   - é um comando que **localiza processos ativos e retorna seus identificadores de processo (PIDs)** a partir de um padrão de nome ou outros critérios, funcionando como uma forma mais simples e direta do que usar `ps aux | grep nome` para encontrar PIDs de processos específicos, como `pgrep firefox`, que exibiria os PIDs de todos os processos do firefox em execução.

```bash

pgrep firefox

```

```bash

ps aux | grep ssh

```

### Finalizar processos

1. `kill <PID>`
   - envia um **sinal amigável** para o programa fechar (SIGTERM).
   - fecha o processo preservando dados (ele tenta salvar o que pode antes de sair).
  
```bash

# primeiro você acha o PID (ex: 1234)
pgrep firefox

# depois envia o sinal padrão (SIGTERM)
kill 1234

```

2. `kill -9 PID`
   - força o encerramento imediato (SIGKILL) - utiliza a **força bruta**.
   - utilizado quando o processo ignora o *kill* comum.

```bash

# força a interrupção imediata (SIGKILL)
kill -9 1234

```

3. `pkill`
   - mata o processo pelo nome, sem precisar descobrir o PID.
   - encerra processos pelo nome, economizando tempo.

```bash

# fecha todas as instâncias do Chrome
pkill chrome

```

4. `killall`
   - semelhante ao `pkill`, porém mais restritivo, assegurando que todos os processos com o nome exato especificado sejam finalizados.
   - limpa o terminal de vários scripts que ficaram rodando por erro.


```bash

# mata todos os processos chamados 'python3'
killall python3

```
