# Análise de Logs

### Logs do systemd

1. `journalctl`
   - exibe os logs do sistema gerenciados pelo systemd.

```bash

journalctl
# mostra todos os registros disponíveis

```

2. `journalctl -xe`
   - mostra **logs recentes** com detalhes e mensagens de erro.
   - **x**: mensagens explicativas
   - **e**: vai direto para o final do log

```bash

journalctl -xe

```

3. `journalctl -u ssh`
   - exibe os logs relacionados ao serviço ssh.

```bash

journalctl -u ssh
# mostra registros do serviço ssh

```

4. `journalctl -f`
   - acompanha os logs do sistema em tempo real.
   - **f**: follow (seguir)

```bash

journalctl -f
# exibe novos logs conforme são gerados

```

#

### Logs tradicionais

1. `/var/log/syslog`
   - arquivo que registra eventos gerais do sistema.

```bash

less /var/log/syslog
# visualiza os logs do sistema

```

```bash

tail -f /var/log/syslog
# exibe as últimas linhas do arquivo syslog e acompanha novas mensagens em tempo real
# monitora eventos do sistema conforme são registrados

```

```bash

grep -i "error" /var/log/syslog
# filtra e exibe linhas do arquivo syslog que contenham a palavra “error”, ignorando diferenças entre maiúsculas e minúsculas.
# mostra mensagens de erro no syslog
# i: ignore case (não diferencia maiúsculas de minúsculas)

```

2. `/var/log/auth.log`
   - arquivo que registra eventos de autenticação e segurança.

```bash

less /var/log/auth.log
# visualiza tentativas de login e uso de sudo

```

```bash

head -n 20 /var/log/auth.log
# mostra apenas as primeiras 20 linhas de log.

```

3. `/var/log/kern.log`
   - arquivo que registra mensagens e eventos do kernel do sistema.

```bash

less /var/log/kern.log
# visualiza logs relacionados ao kernel

```
