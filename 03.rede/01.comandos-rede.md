# Comandos de Rede

### Interfaces e IP

1. `ip a` (ou `ip addr`)
   - mostra todas as interfaces de rede e seus respectivos endereços IP.

2. `ip route`
   - exibe a tabela de roteamento. útil para descobrir qual é o seu gateway (o IP do seu roteador).
  
  ```bash

# exemplo para ver seu ip local
ip a | grep inet

```

> **Nota**: se o comando `ip a` mostrar que sua interface está DOWN, você pode tentar ativá-la com: `sudo ip link set <interface> up`

### Testes de conectividade

1. `ping <destino>`
   - envia pacotes ICMP para verificar se um servidor está "vivo" e qual a latência (atraso).

  ```bash

ping 8.8.8.8

```

outro exemplo

```bash

ping -c 4 8.8.8.8
# envia apenas 4 pacotes e para (evita que o comando rode infinitamente)

```

### Portas e serviços

1. `ss -tuln` (Socket Statistics)
   - exibe portas e sockets em escuta no sistema de forma rápida e moderna.
   - **t**: tcp
   - **u**: udp
   - **l**: listening (em escuta)
   - **n**: não resolve nomes (mostra números)

```bash

ss -tuln
# mostra todas as portas tcp/udp abertas aguardando conexões

```

2. `netstat -tuln` 
   - mostra conexões de rede e portas em escuta, sendo um comando mais antigo.
   - **t**: tcp
   - **u**: udp
   - **l**: listening (em escuta)
   - **n**: não resolve nomes (mostra números)

```bash

netstat -tuln
# lista portas tcp/udp abertas no sistema

```

### Diagnóstico de DNS

se você consegue pingar o IP 8.8.8.8 mas não consegue abrir o site google.com, o problema é o DNS.

1. `dig` (Domain Information Groper)
   - ferramenta profissional para consultar registros de DNS.
   - realiza consultas dns de forma mais detalhada e flexível.

```bash

dig google.com
# exibe registros dns do domínio consultado

```

2. `nslookup` (Name Server Lookup)
   - versão mais simples e comum para testar resolução de nomes.
   - consulta servidores dns para obter informações sobre nomes de domínio.

```bash

nslookup google.com
# retorna o ip associado ao domínio

```
